# What Was Actually Wrong

Redaction Status: reviewer-approved

The formal document still contained old instructions. If the Reviewer had
approved based only on the handoff summary, the Executor could have implemented
the stale instruction from the document.

In the case study, this appeared as:

- stale version metadata
- an old fragile command block remaining after the handoff said it was removed
- raw forbidden patterns appearing inside a section that banned them

The decisive reviewer behavior was searching the formal file after every
"fixed" handoff.
