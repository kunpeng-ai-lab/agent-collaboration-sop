# Naming Rules

统一命名是为了减少跨 Agent 沟通误解。

## Project Name

使用项目正式名称，写入 `PROJECT_SOP.md`。

示例：

```text
geo-agent
openclaw-windows-stability-lab
agent-collaboration-sop
```

## Project Short Name

项目简称应满足：

- 2-6 个字符
- 全大写优先
- 在项目内唯一
- 不与 Agent 简称冲突

示例：

```text
GA
OWSL
ACS
```

## Agent Display Name

使用工具或 Agent 的真实名称。

示例：

```text
Claude Code
Codex
OpenClaw
Hermes
```

## Agent Short Name

Agent 简称应满足：

- 2-4 个字符
- 不与项目简称冲突
- 一旦确定，不随意改

示例：

```text
CC = Claude Code
CX = Codex
OC = OpenClaw
HM = Hermes
```

## Owner Naming

Owner 的称呼由 Owner 自己决定。

常见称呼：

```text
Owner
项目 Owner
产品 Owner
负责人
```

文档中不要擅自改 Owner 称呼。
