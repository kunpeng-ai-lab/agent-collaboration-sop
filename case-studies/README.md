# ACS Case Studies

This directory stores sanitized standard collaboration cases.

Case studies teach good Executor/Reviewer/Owner behavior through real or
realistic phase examples.

## Required Files Per Case

Use one directory per case:

```text
case-studies/<case-id>/
  case-study-summary.md
  reviewer-lessons.md
  executor-lessons.md
  evidence-patterns.md
  checklist-updates.md
```

Optional:

```text
conversation-excerpts.md
```

Conversation excerpts must be minimized and redacted.

## Redaction

Before committing a case, follow:

```text
docs/case-study-redaction.md
```

Never commit private customer names, credentials, local paths, private repo
URLs, raw screenshots, or unreleased commercial details.

## Case Study Purpose

Each case should answer:

- What was the phase goal?
- What did Executor do?
- What did Reviewer check?
- What did Owner decide?
- What mistake or risk was caught?
- What ACS rule changed or became clearer?
- What should future teams copy?
- What should future teams avoid?
